## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Remove Blesta entirely | User uses GHL for all client/CRM management |
| Next.js App Router | Server-side API routes secure API keys without a separate Express server |
| Navigation shim (`src/lib/navigation.js`) | Allows existing components to work without rewriting every Link/useNavigate call |
| Rename `src/pages/` → `src/views/` | Avoids Next.js Pages Router conflict |
| Firebase Auth stays | Handles authentication; GHL handles CRM profile |
| HostAfrica WHMCS API | Domain + VPS provisioning via reseller API |
| n8n stays | Email automation + webhooks, triggered by GHL pipeline changes |
| PayFast stays | Payment gateway for ZAR transactions |
